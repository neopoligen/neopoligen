Getting Started With Neopoligen

An app for building websites

Neopoligen is a website builder. Its goal is to make it as easy as possible for anyone to make a website for free. It works with files you write on your computer that look like this: 

-- title
-- subtitle: An app for building websites

Content Files

Neopoligen keeps your original content files directly on your computer. They're in a "Neopoligen" folder under your main 'Documents' folder. The app can make multiple sites so each one gets its own folder. When you first install the app, it makes a new site in the 'Example Site' folder which you can see here:

The initial contents of a site folder are:

  • config.json file - Contains some basic details about the site. (Details on the configuration can be found in the documentation)
  • configuration folder - This also holds some configuration items, but it's going away. Everything is moving into the config.json file
  • content folder - This is where all the main content files that make up your site go. The files all end with a .neo extension (if you can see those in your finder) which is the type of file that Neopoligen uses.
  • docs folder - This is where the files that make up the actual website are output when Neopoligen builds them. You don't need to mess with it, but feel free to poke around to see what the output is.
  • images folder - For any images you want to include on your site. You can organize images however you want inside this folder and then include them in on a page using just the name of the file. More details in the documentation)
  • themes folder - Neopoligen works by combining your content files with a theme to build the site. This is where they are stored.

Sections

Neopoligen files are based of Sections. Each one starts with two dashes (i.e. "--") followed by the type of section and each one belong to a Category of section types that helps determine it's behaviour.

The two sections we've seen so far are -- title and -- metadata. The title section is part of the Standard category which generally outputs directly to the page. The metadata section is part of a category called JSON. Sections in the JSON category don't show anything by default. They can be made to by applying a template, but we're not outputting anything for metadata. It's merely a way to pass the Date and the ID that Neopoligen uses for making links in the page.

Section Attributes

The way that we're passing the Date and ID is with Section Attributes which are lines directly after the start of a section that also start with two dashes. The metadata section on the Example-Site home-page.neo example looks like this: 

-- metadata
-- date: 2024-03-11
-- id: 2dwmgthu
-- path: /

The -- metadata line starts the section and the -- date, -- id, and -- path lines are all attributes. (If you're wondering, the -- path attribute that has the / after it is what's used to identify the home page of the site. More details on that in the documentation.)

The example theme doesn't do anything visible with those attributes but we can use the -- title section to see something change. Add a line with -- class: green underneath the -- title line so it looks like this: 

-- title
-- class: green

Neopoligen Example Site

When you save the page you'll see green text show up in the browser.

Adding a -- class attribute to a section tells it to change its formatting based of what are called Stylesheets. If you look in the folder:

Example-Site / themes / example-site / _assets

You'll see a file called

styles.css

If you click on styles.cssin the sidebar of Visual Studio Code to open it and scroll down some you'll see this snippet which is what does the color change:

.green {
  color: green;
}

The full name for the way styles work is called Cascading Style Sheets (or, CSS for short). You can do a lot with them. If you're interested, you can get started learning more about them here: CSS basics

Custom Section Attributes

Attributes can be used for more than just changing the style of a section. They can also be used to send content to the templates for formatting. For example: add a -- quote section above the -- metadata section with something like: 

-- quote 

Hello, World!

When you save the file in Visual Studio Code it'll show up in the browser. In this case, the template that renders quote sections in the theme has already applied a style that added the border to the quote. Now let's add some more formatted content with a -- by attribute like this:

-- quote 
-- by: Alan

Hello, World!

When you save the file you'll see a new attribution show up under the quote. We can also add a -- source like: 

-- quote 
-- by: Alan
-- source: The Internet

Hello, World!

If you want to see how the template the theme is using to output the quote it's in:

Example-Site / themes / example-site / sections / quote / full / default.jinja

There's more details on template and themes in the documentation if you'd like to experiment modifying one or making your own.

Spans

The last thing to know about to get started is Spans. They're used for formatting content inside a section. For example, change the Hello, world in your quote to this:

Hello, <<strong|world>>

When you save the file the world "world" changes to bold on the page

You can add styles to spans as well. Try this: 

Hello, <<strong|world|class: green>>

Spans are also how linking is done. There's a ilink (for internal link) span that uses an ID to link between pages. For example, the second-page.neo file has an id of 2dwyp9ty. You can use that to make a link to it from the home-page.neo file with:

<<ilink: 2dwyp9ty|Link To Second Page>>
